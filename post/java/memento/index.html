[java] Memento 패턴

0001-01-01
java

메멘토 패턴은 캡슐화를 유지하면서 상태를 저장/복원할 수 있는 패턴이다.
즉, 원하는 시점의 상태 복원이 가능해진다.

구조
Originator : 현재 상태를 저장하고 Memento 객체의 정보를 관리한다.
Memento : 특정 시점의 상태를 저장하는 클래스.
Caretaker : 현재의 Originater 역할의 상태를 저장.

public class Originator {
    private String state;
    
    public String getState() {
        return state;
    }
    
    public void setState(String state) {
        this.state = state;
    }
    
    public Memento save(){
        return new Memento(state);
    }
    
    public void restore(Object obj){
        this.state = ((Memento) obj).state;
    }
    
    private class Memento {
        private final String state;

        public Memento(String state) {
            this.state = state;
        }
    }
}

public class Main {
    public static void main(String[] args){
        List<Object> caretaker = new ArrayList<>();
    
        Originator originator = new Originator();
        originator.setState("memento #1");
        caretaker.add(originator.save());
        originator.setState("memento #2");
        caretaker.add(originator.save());
        
        originator.restore(caretaker.get(0));
        System.out.println(originator.getState());
        originator.restore(caretaker.get(1));
        System.out.println(originator.getState());
    }
}
  • Memento : 상태를 나타내는 클래스. Originator 객체만 접근할 수 있게 만듦.
  • Originator : Memento의 인스턴스를 만든다.

Memento 객체를 자주 생성하는 경우 메모리를 많이 사용할 수 있고, 사용되지 않는 메모리를 추적해야 한다.
