[java] Facade 패턴

0001-01-01
java

퍼사드 패턴은 구조 패턴의 한 종류이다. 설계의 목표는 서브시스템 간 종속성을 최소화하는 것.
서브시스템에 대한 의존성을 한곳으로 모을 수 있는 장점이 있지만, 클라이언트가 서브시스템을 직접 사용하는 것을 막을 수 없다.

예제) 웹 서버의 종류는 한 가지라고 가정. 웹 서버와 WAS를 원하는 개수만큼 설정하고 입력된 url로 html을 받는다.

public class WebServer {
    private String ip;
    private int port;
    private Application[] was;
    
    public WebServer(String ip, int port, Application[] was) {
        this.ip = ip;
        this.port = port;
        this.was = was;
    }
    
    public String getHtml(String path){
        String html = "";
        // 동적 페이지는 데이터 베이스 및 서버에서 웹 페이지 작업을 처리한다.
        if(was != null && was.length > 0 && path.indexOf(".do") > -1){
            double dValue = Math.random();
            int iValue = (int)(dValue * was.length);
            
            Application target = was[iValue];
            
            target.connection();
            html = "HTML";
        }
        
        return html;
    }
}

public class WebFacade {
    private WebServer[] webServers;
    
    public WebFacade(WebServer[] webServers) {
        this.webServers = webServers;
    }
    
    public String html(String url){
        String html ="";
        if(webServers != null){
            String hostname = url;
            String ip = new DNS().convert(hostname);
            // hostname 으로 얻은 ip 주소로 웹 서버를 찾는다. (예시) 랜덤으로..
            double dValue = Math.random();
            int target = (int)(dValue * webServers.length);
            WebServer webServer = webServers[target];
            
            String path = url;
            html = webServer.getHtml(path);
        }
        
        return html;
    }
}

public class Main {
    public static void main(String[] args){
        WebServer[] webServers = new WebServer[]{
            new WebServer("192.168.0.1", 80, new Application[]{new JavaServer(), new JavaServer(), new NodeServer()}),
            new WebServer("192.168.0.2", 80, new Application[]{new NodeServer(), new NodeServer(), new JavaServer()})
        };
        WebFacade webFacade = new WebFacade(webServers);
        webFacade.html("https://~.do");
    }
}
  • WebFacade : 퍼사드 객체는 모든 로직을 처리한다. 클라이언트에서 이 클래스만 알고 있으면 html을 가져올 수 있다.
  • 클라이언트는 DNS, WebServer, Application 객체를 알고 있지 않아도 퍼사드 객체 하나만으로 동작을 할 수 있다.

웹 동작 순서
1.웹 브라우저의 검색창에 URL을 입력한다.
2. 브라우저가 DNS에게 특정 사이트의 주소를 요청한다.
3. DNS가 웹 브라우저에게 사이트의 IP주소를 응답한다.
4. 웹 브라우저가 웹 서버에게 IP주소를 이용하여 html를 요청한다.
5. URL정보에 해당하는 데이터를 검색하고 처리한다.
6. WAS에서 처리된 결과를 웹 서버로 보낸다.
7. 웹 서버는 웹 브라우저에게 html을 응답한다.
8. 웹 브라우저는 화면에 html을 출력한다.

서브 클래스가 변경되어도 클라이언트는 퍼사드 객체만 참조하므로, 코드에 영향이 가지 않는다.
