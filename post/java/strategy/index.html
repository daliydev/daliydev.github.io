[java] Strategy 패턴

0001-01-01
java

전략 패턴은 교체 가능한 알고리즘을 만드는데 사용되는 패턴이다.
동일 계열의 알고리즘을 정의하고, 각 알고리즘을 캡슐화하여 상호교환이 가능하도록 만든다.
단일 상속만 가능한 자바에서는 템플릿 메서드보다는 전략 패턴이 더 많이 활용된다.

예제) 문서와 파일을 다른 시스템에 공유하려고 한다.

public interface ShareStrategy {
    public void send();
}

public class KaKaoStrategy implements ShareStrategy {
    
    @Override
    public void send() {
    }
}

public class LineStrategy implements ShareStrategy {
    
    @Override
    public void send() {
    }
}

public class Sharing {
    private ShareStrategy shareStrategy;
    
    public void setStrategy(ShareStrategy shareStrategy) {
        this.shareStrategy = shareStrategy;
    }
    
    public void share(){
        shareStrategy.send();
    }
}

// 문서 객체
public class Document extends Sharing {
}
// 파일 객체
public class File extends Sharing {
}

public class Main {
    public static void main(String[] args){
        Sharing document = new Document();
        Sharing file = new File();
        
        // 문서 라인 공유
        document.setStrategy(new LineStrategy());
        document.share();
        // 파일 카카오 공유
        file.setStrategy(new KaKaoStrategy());
        file.share();
    }
}
  • ShareStrategy : 전략 패턴을 사용하기 위한 인터페이스
  • KaKaoStrategy, LineStrategy : ShareStrategy을 실제 구현하는 클래스
  • Sharing : 전략의 메서드를 호출해서 사용하는 클래스
  • Strategy 인터페이스가 변화에 대해 Sharing 클래스의 변경을 차단해 준다. 즉, 기존의 코드 변경을 최소화하면서 확장할 수 있으므로 OCP에 만족하는 설계이다.

단점은 각 전략은 인터페이스 객체로 생성되기 때문에 전략의 개수만큼 객체가 생성되어야 한다.
