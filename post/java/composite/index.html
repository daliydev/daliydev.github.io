[java] Composite 패턴

0001-01-01
java

컴포지트 패턴은 클라이언트가 복합 객체나 단일 객체를 동일하게 취급하는 것을 목적으로 한다.
전체 구조를 나타내기 위해 객체를 트리 구조로 구성합니다.

예제) 이동 수단을 예약하는 프로세스를 예제로 사용하겠습니다.

public interface Transportation {
    public void reservation();
}

public class Bus implements Transportation {
    
    @Override
    public void reservation() {
        System.out.println("bus reservation");
    }
}

public class Airplane implements Transportation {
    
    @Override
    public void reservation() {
        System.out.println("airplane reservation");
    }
}

import java.util.ArrayList;
import java.util.List;

public class Composite implements Transportation {
    
    private List<Transportation> transportationList = new ArrayList<>();
    
    @Override
    public void reservation() {
        for(Transportation transportation : transportationList){
            transportation.reservation();
        }
    }
    
    public void add(Transportation transportation){
        transportationList.add(transportation);
    }
    
    public void remove(Transportation transportation){
        transportationList.remove(transportation);
    }
}

public class Main {
    public static void main(String[] args){
        Transportation transportation1 = new Bus();
        Transportation transportation2 = new Airplane();
        
        Composite composite = new Composite();
        composite.add(transportation1);
        composite.add(transportation2);
        
        composite.reservation();
        // bus reservation
        // airplane reservation
    }
}
  • Transportation 인터페이스 역할
  • Bus, Airplane 지정된 인터페이스를 구현할 클래스
  • Composite 클래스는 Transportation 요소를 자식으로 가지고, 요소를 관리하기 위한 메서드를 구현함

구성 패턴을 적용하지 않는다면, 새로운 객체가 추가될 때마다 Composite 클래스가 변경될 수 있어서 OCP 규칙에 맞지 않다.
