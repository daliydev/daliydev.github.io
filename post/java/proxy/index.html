[java] Proxy 패턴

0001-01-01
java

프록시 패턴은 사용할 객체를 직접적으로 참조하는 것이 아닌 중간 객체를 통해 대상에 접근하는 방법이다.
단순히 보안상의 이유만으로 사용하는 것이 아닌, 캐싱 처리함으로써 부하를 줄일 수도 있다.
하지만 프록시 객체를 사용하면 한 단계를 더 거치는 작업이므로 응답이 느려질 수 있다.

예시) 결제 수단에 따른 처리 > 신용카드에 대해 최소 결제 금액을 두어 객체(CreditService) 접근을 제한한다.

public class PaymentProxy implements PaymentExecutor {
    
    private int price;
    private PaymentExecutor paymentExecutor;
    
    public PaymentProxy(int price, PaymentExecutor paymentExecutor) {
        this.price = price;
        this.paymentExecutor = paymentExecutor;
    }
    
    @Override
    public void run() {
        if(paymentExecutor instanceof CreditService){
            if(price < 50000){
                throw new RuntimeException("50,000원 미만은 신용카드 결제가 불가능합니다.");
            }
        }
        
        paymentExecutor.run();
    }
}

public class Main {
    public static void main(String[] args){
        PaymentProxy proxy = new PaymentProxy(40000, new BankTransferService());
        proxy.run(); // bank transfer
        proxy = new PaymentProxy(40000, new CreditService());
        proxy.run(); // exception
    }
}
  • BankTransferService, CreditService 기능을 수행할 객체
  • PaymentExecutor 동일하게 다룰 인터페이스
  • PaymentProxy 클라이언트와 기능 수행할 객체 사이에 존재하는 객체
