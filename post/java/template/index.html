[java] Template 패턴

0001-01-01
java

템플릿 메서드 패턴은 공통으로 사용하는 메서드는 상위 클래스에서 정의하고, 다르게 구현할 메서드는 추상화하여 하위 클래스에서 구현하도록 하는 패턴이다.
코드 중복을 최소화하고 재사용성이 증가한다.

각 클래스는 아래와 같은 역할을 한다.

  • AbstractClass
    템플릿 메서드를 정의하는 클래스
    공통 메서드를 정의하고, 하위 클래스에서 구현될 기능을 primitive 메서드 또는 hook 메서드로 정의하는 클래스
  • ConcreteClass
    물려받은 primitive 메서드 또는 hook 메서드를 구현하는 클래스
    상위 클래스에 primitive/hook 메서드를 오버라이드 하여 구현하는 클래스
예제) 주문을 하는 과정을 나타낸 것이다.

public abstract class Order {
    
    public void run(){
        try {
            db();
            paymentAPI();
        }catch (Exception e){
            throw new RuntimeException();
        }
    }
    
    private void db(){
    }
    
    abstract void paymentAPI();
    abstract Object getPayment(String id);
}

public class InicisAPI extends Order {
    
    @Override
    void paymentAPI() {
        System.out.println("inicis payment request");
    }
    
    @Override
    Object getPayment(String id) {
        return null;
    }
}

public class DanalAPI extends Order {
    
    @Override
    void paymentAPI() {
        System.out.println("danal payment request");
    }
    
    @Override
    Object getPayment(String id) {
        return null;
    }
}

public class Main {
    public static void main(String[] args){
        DanalAPI danalAPI = new DanalAPI();
        danalAPI.run();
        InicisAPI inicisAPI = new InicisAPI();
        inicisAPI.run();
    }
}
  • Order : AbstractClass를 나타내며 공통 메서드를 정의한다.
  • InicisAPI, DanalAPI : ConcreteClass를 나타내며 각 결제사마다 다른 연동 방식일 때 하위 클래스에서 구현한다.

전체적인 틀은 통일 시키지만, 추상화된 메서드로 유연하게 기능을 변경할 수 있도록 하는 디자인 패턴이다.
