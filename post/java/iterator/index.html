[java] Iterator 패턴

0001-01-01
java

이터레이터 패턴은 객체를 저장하는 방식은 보여주지 않으면서, 모든 항목에 접근할 수 있게 해준다.
반복 작업을 Iterator 인터페이스를 이용하여 캡슐화한다.

구조
Iterator 순서대로 검색해가는 인터페이스. hasNext()와 next()를 정의
ConcreateIterator Iterator를 구현함. 검색하기 위해 필요한 정보를 가지고 있어야 함.
Aggregate Iterator 역할을 만들어내는 인터페이스.
ConcreateAggregate Aggregate 인터페이스 구현 클래스. 구체적인 역할.

public class Iterator {
    private Aggregate aggregate;
    private int index = 0;
    
    public Iterator(Aggregate aggregate) {
        this.aggregate = aggregate;
    }
    
    public boolean hasNext() {
        return index < aggregate.length() ? true : false;
    }
    
    public Object next() {
        Object object = aggregate.childAt(index);
        index++;
        return object;
    }
}

public interface Aggregate {
    Iterator iterator();
    int length();
    Object childAt(int index);
}

public class Cart implements Aggregate {
    private Goods[] goodsArray;
    private int index = 0;
    public Cart(int size) {
        this.goodsArray = new Goods[size];
    }
    
    public void add(Goods goods){
        this.goodsArray[index] = goods;
        index++;
    }
    
    @Override
    public int length() {
        return goodsArray.length;
    }
    
    @Override
    public Object childAt(int index) {
        return goodsArray[index];
    }
    
    @Override
    public Iterator iterator() {
        return new Iterator(this);
    }
}

public class Goods {
    private String name;
    private String brand;
    
    public Goods(String name, String brand) {
        this.name = name;
        this.brand = brand;
    }
    
    public String getName() {
        return name;
    }
}

public class Main {
    public static void main(String[] args){
        Cart cart = new Cart(4);
        cart.add(new Goods("test1", ""));
        cart.add(new Goods("test2", ""));
        cart.add(new Goods("test3", ""));
        cart.add(new Goods("test4", ""));
        
        Iterator iterator = cart.iterator();
        while(iterator.hasNext()){
            Goods c = (Goods) iterator.next();
            System.out.print(c.getName()); // test1 test2 test3 test4 
        }
    }
}
  • Iterator : 생성자에 Aggregate 인자를 받아서 인터페이스 없이 처리함.
  • Aggregate : 역할을 만드는 인터페이스로, Iterator에서 공통으로 호출할 메서드를 정의함.
  • Goods : 집합체의 요소가 되는 클래스.
  • Cart : 집합체, Aggregate가 선언한 인터페이스를 구현하는 클래스.
  • 위 while 루프는 Cart의 구현에는 의존하지 않는다.

Iterator은 모든 Collection의 하위 클래스에서 사용 가능한 게 장점이지만, Collection 크기가 크면 성능 저하가 온다.
