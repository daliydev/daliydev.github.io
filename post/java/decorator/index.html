[java] Decorator 패턴

0001-01-01
java

데코레이터 패턴은 어떤 객체에 책임을 동적으로 추가하는 패턴입니다.
즉, 객체에 새로운 기능을 추가하거나 변화시켜서 기능을 유연하게 확장할 수 있게 해줍니다.
단점은 의미 없는 객체가 많아질 수 있고, 때문에 코드가 복잡해짐

예제) 진공 간에 산소/질소 물질을 추가한다. 물질이 추가될 때마다 Decorator를 상속받는 클래스를 만든다.

public interface Space {
    String add();
}

public class BaseSpace implements Space {
    
    @Override
    public String add() {
        return "vacuum";
    }
}

public class SpaceDecorator implements Space {
    private Space space;
    
    public SpaceDecorator(Space space) {
        this.space = space;
    }
    
    @Override
    public String add() {
        return space.add();
    }
}

public class Oxygen extends SpaceDecorator {
    
    public Oxygen(Space space) {
        super(space);
    }
    
    @Override
    public String add() {
        return super.add() + " oxygen";
    }
}

public class Nitrogen extends SpaceDecorator {
    
    public Nitrogen(Space space) {
        super(space);
    }
    
    @Override
    public String add() {
        return super.add() + " nitrogen";
    }
}

public class Main {
    public static void main(String[] args){
        Space space = new Oxygen(new Nitrogen(new BaseSpace()));
        System.out.println(space.add()); // vacuum nitrogen oxygen
    }
}
  • Space : 기능의 인터페이스
  • BaseSpace : 실제로 구현하는 역할(기본 기능)
  • SpaceDecorator : 구체적인 Decorator의 공통 기능을 제공
  • Oxygen, Nitrogen 클래스는 SpaceDecorator의 하위 클래스로 개별적인 기능
